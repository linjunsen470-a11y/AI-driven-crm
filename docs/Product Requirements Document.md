# AI 语音 CRM — 产品需求文档

## 1. 项目概述

构建一个**极简 AI 驱动的 CRM 系统**，专为频繁与客户沟通的销售团队设计。

系统采用**语音优先工作流**：

销售人员在与客户沟通后上传语音录音，系统自动：

1. 转录语音录音
2. 生成跟进摘要
3. 提取结构化客户信息
4. 自动更新 CRM 记录

目标：最小化手动数据录入，最大化信息捕获。

---

## 2. 核心产品理念

传统 CRM 需要手动输入数据。

本系统反转工作流：

| 销售人员操作 | 系统责任 |
|-------------|---------|
| 上传语音笔记 | 转录 |
| | 摘要生成 |
| | 标签分类 |
| | CRM 记录创建 |

**主要用户：**

销售代表，其特征：

| interest_level | enum | 意向等级 |

| updated_at | datetime | 更新时间 |

## 6. 语音识别

使用语音转文字引擎：

**可选方案：**

- Whisper
- whisper.cpp
- OpenAI Whisper API

**输出示例：**

```
客户计划两年后养老
预算大约 50 万
子女在上海
```

保存至：`followups.transcription`

---

## 7. AI 摘要生成

LLM 生成对话摘要。

**输入：**

转录文本

**Prompt 示例：**

```
请总结以下客户对话。

提取关键点：

* 客户意向
* 时间线
* 财务预算
* 核心关注点
```

**输出示例：**

```
客户计划两年后养老
预算约 50 万
对子女支持养老决策
对康养环境有兴趣
```

保存至：`followups.ai_summary`

---

|------|---------|

# 📋 优化建议

## 一、架构与技术层面

### 1.1 异步任务队列
- **问题**：AI 处理（转录 + LLM）可能耗时 10-60 秒，同步处理会阻塞请求
- **建议**：引入任务队列（Bull/Redis）实现异步处理
- **新增字段**：`followups.processing_status` (pending/processing/completed/failed)

### 1.2 错误处理与重试机制
- **问题**：AI API 可能失败，需考虑重试策略
- **建议**：
  - 添加失败重试机制（最多 3 次）
  - 失败时通知用户
  - 记录错误日志

### 1.3 文件存储策略
- **建议明确**：
  - 语音文件保留策略（如自动删除 90 天前文件）
  - 文件大小限制（如最大 50MB）
  - 转码策略（统一转为 mp3 便于处理）

## 三、AI 能力增强

### 3.1 多语言支持
- 考虑客户可能使用方言或英语
- 建议：Whisper 支持多语言，需明确语言检测策略

### 3.2 置信度评分
- AI 提取的字段添加置信度评分
- 低置信度字段标记为"待确认"，提醒销售复核

### 3.3 增量更新策略
- 当新跟进与已有信息冲突时（如预算变化）
- 策略：以最新跟进为准，或标记为"信息变更"

### 3.4 Prompt 工程优化
- 建议添加 few-shot examples（示例对话）
- 针对养老行业定制专业术语

## 四、用户体验优化

### 4.1 语音上传体验
- 支持**浏览器直接录音**（无需外部录制）
- 显示上传进度条
- 支持批量上传

### 4.2 跟进时间线增强
- 按时间倒序展示
- 支持快速筛选（仅看语音/仅看手动备注）
- 支持在时间线上直接编辑 AI 摘要

### 4.3 客户列表增强
- 添加搜索功能（按姓名/电话/标签）
- 添加筛选（按意向等级/城市/最后跟进时间）
- 添加排序（按最后跟进时间/意向等级）

### 4.4 提醒功能
- 根据 `next_action` 自动生成待办提醒
- 逾期跟进提醒（如 7 天未联系）
