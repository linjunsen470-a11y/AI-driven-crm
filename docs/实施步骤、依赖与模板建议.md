# AI CRM 实施步骤、依赖与模板建议

这份文档的目标是回答 4 个问题：

1. 你下一步应该先做什么
2. 哪些依赖是必须装的
3. 哪些模板适合你，哪些不适合
4. 你应该用什么顺序把系统真正落地

---

## 1. 我的总体建议

如果你现在要开始做，我最推荐的路线不是：

- 直接找一个重型 CRM 二开
- 或一上来就把所有 AI 能力都做全

而是：

`用官方 Next.js 脚手架起项目 + 用 shadcn/ui 搭工作台 + 用 Prisma 管 PostgreSQL + 用 n8n 编排异步任务 + 用本地文件系统/MinIO 存海量文件`

这是目前最平衡的方案，因为它兼顾了：

- 开发速度
- 后期可维护性
- AI 工作流可扩展性
- 本地部署和内部使用的现实性

---

## 2. 模板怎么选

## 2.1 最推荐的模板组合

### 前端脚手架

推荐：

- `create-next-app`

原因：

- 官方维护
- 起步最稳
- App Router、TypeScript、Tailwind、Lint 都能直接带上
- 不会引入你不需要的业务包袱

### UI 模板

推荐：

- `shadcn/ui`

原因：

- 很适合内部工作台和管理后台
- 组件透明，可控，不是黑盒
- 和 Next.js 配合成熟
- 后续做客户详情页、表格、抽屉、对话框、表单非常顺手

### 不推荐的模板

不建议：

- 重型开源 CRM 模板
- 大而全 SaaS Starter
- 带计费、多租户、营销站、博客的全家桶模板

原因：

- 你的差异化在 AI CRM 工作流，不在那些通用 SaaS 配件
- 这类模板会拖慢结构收敛

---

## 2.2 我建议你怎么起项目

### 推荐命令

```bash
pnpm create next-app
```

推荐选择：

- TypeScript
- ESLint 或 Biome 二选一
- Tailwind CSS
- App Router
- `src/` 目录
- 默认导入别名 `@/*`

如果你想更直接，也可以显式使用 `create-next-app`。

参考来源：

- Next.js 官方 `create-next-app` 文档说明了 CLI 用法和可选项：<https://nextjs.org/docs/app/api-reference/create-next-app>
- Next.js 官方安装文档说明默认推荐配置包含 TypeScript、Tailwind、App Router 等：<https://nextjs.org/docs/app/getting-started/installation>

### UI 初始化

```bash
pnpm dlx shadcn@latest init -t next
```

参考来源：

- shadcn/ui 官方 Next.js 安装文档：<https://ui.shadcn.com/docs/installation/next>

## 3. 推荐依赖清单

下面按“必须 / 强烈推荐 / 可后置”来分。

## 3.1 必须安装

### 数据库与 ORM

```bash
pnpm add @prisma/client
pnpm add -D prisma
```

原因：

- 你已经明确要用 PostgreSQL
- Prisma 很适合你这种“表结构不断演进”的项目
- 和 Next.js、TypeScript 配合很顺

参考来源：

- Prisma 官方 Next.js 指南：<https://www.prisma.io/docs/guides/nextjs>
- Prisma 官方 PostgreSQL 支持文档：<https://docs.prisma.io/docs/orm/core-concepts/supported-databases/postgresql>

### 表单与校验

```bash
pnpm add react-hook-form zod @hookform/resolvers
```

原因：

- 你的“确认建档表单”会很重
- `react-hook-form` 非常适合大表单和低重渲染
- `zod` 很适合把 AI 输出、API 输入和表单校验统一起来

参考来源：

- React Hook Form 官方站点：<https://react-hook-form.com/>
- Zod 官方文档：<https://zod.dev/>

### 基础 UI

至少会频繁用到这些 shadcn 组件：

- button
- input
- textarea
- select
- checkbox
- badge
- card
- dialog
- drawer
- table
- tabs
- form
- toast / sonner

### 日期处理

```bash
pnpm add date-fns
```

原因：

- 跟进时间、提醒时间、参观日期、入住日期都需要格式化和计算

参考来源：

- `date-fns` npm 说明：<https://www.npmjs.com/package/date-fns>

## 3.2 强烈推荐安装

### 请求状态管理

```bash
pnpm add @tanstack/react-query
```

原因：

- 你会有很多列表和详情接口
- `interactions` 状态轮询、提醒列表、客户详情都适合它

参考来源：

- TanStack Query 官方安装文档：<https://tanstack.com/query/latest/docs/react/installation>

### 表格能力

```bash
pnpm add @tanstack/react-table
```

原因：

- 客户列表、待确认队列、提醒列表都适合 TanStack Table
- 和 shadcn 组合很好用

参考来源：

- TanStack Table 文档：<https://tanstack.com/table/latest/docs/framework/react/react-table>

### 图标

```bash
pnpm add lucide-react
```

原因：

- 和 shadcn 搭配最顺
- 内部工作台足够用了

### 日志

```bash
pnpm add pino
```

原因：

- AI 处理链路和文件处理很需要结构化日志
- 后面查错会省很多时间

## 3.3 如果你第一版就要做登录

我推荐：

```bash
pnpm add better-auth
```

原因：

- 和 Next.js 集成直接
- 适合你这种内部系统先做基本账号体系

参考来源：

- Better Auth 官方安装文档：<https://better-auth.com/docs/installation>
- Better Auth 官方 Next.js 集成文档：<https://www.better-auth.com/docs/integrations/next>

如果你第一版只是内部演示，也可以暂时不接登录，先做假用户或简单保护。

## 4. 后端和基础设施怎么落地

## 4.1 本地运行推荐形态

推荐使用 Docker Compose 起下面几项：

- PostgreSQL
- n8n
- MinIO
- Redis

### 为什么这样配

`PostgreSQL`

- 存主数据、AI 状态、提醒、聊天会话、知识元数据

`n8n`

- 跑异步流程、定时任务、转录/OCR/提取/提醒生成

`MinIO`

- 存录音、截图、OCR 原图、导出文件
- 兼容 S3 API，后面迁移更容易

`Redis`

- 可选但非常建议
- 后面做队列、缓存、幂等和状态管理会更顺

参考来源：

- n8n 官方 Docker 安装文档：<https://docs.n8n.io/hosting/installation/docker/>
- MinIO/AIStor 官方容器与对象存储文档：<https://min.io/docs/minio/container/index.html>

---

## 4.2 数据库不要存什么

不要直接把这些存进 PostgreSQL 大字段里：

- 原始音频二进制
- 聊天截图二进制
- OCR 图片原件

数据库只存：

- 文件路径
- bucket
- 文件大小
- 哈希
- 所属 interaction
- 处理状态

## 5. 具体落地顺序

## Phase 0：基础环境

目标：

- 确保本地开发能启动

你要完成：

1. 安装 Node.js 20+
2. 安装 `pnpm`
3. 安装 Docker / Docker Compose
4. 起 PostgreSQL、n8n、MinIO
5. 建 `.env` 模板

---

## Phase 1：前端和数据库骨架

目标：

- 先把项目壳子搭好

你要完成：

1. 用 `create-next-app` 初始化项目
2. 初始化 `shadcn/ui`
3. 安装 Prisma，接 PostgreSQL
4. 落你已经整理好的 `schema.prisma`
5. 生成初始 migration
6. 做基础 layout 和导航

建议先出 4 个页面：

- 待确认队列页
- 上传页
- 客户详情页
- AI Chat 占位页

---

## Phase 2：主链路 MVP

目标：

- 跑通 `上传 -> AI 解析 -> 确认建档`

你要完成：

1. 文件上传接口
2. `interactions` 创建接口
3. 处理状态轮询
4. AI 提取结果确认表单
5. 建档/更新客户接口
6. 客户详情页时间线

这一步先不要追求：

- 知识库
- 批量导入录音
- 高级提醒

---

## Phase 3：异步工作流

目标：

- 用 `n8n` 接管耗时任务

你要完成：

1. n8n 监听新 interaction
2. 音频转录
3. 图片 OCR
4. 结构化提取
5. 回写数据库
6. 错误重试和失败标记

这一步完成后，你的系统才真正像一个 AI CRM，而不只是一个前端 Demo。

---

## Phase 4：增强能力

目标：

- 提升自动化和销售实用性

你要完成：

1. 忽略号码表
2. 批量导入通话录音
3. 企业知识库导入
4. 销售 AI Chat
5. 规则驱动的跟进提醒

---

## Phase 5：后置高级能力

目标：

- 让系统越来越智能

你要完成：

1. AI 辅助提醒理由
2. 历史转化数据分析
3. 跟进时机预测
4. 更细粒度知识检索

---

## 6. 我对你“下一步行动”的建议

如果你现在还在评估阶段，我建议你先做下面这件事：

### 最优下一步

先搭一个最小工程骨架，而不是继续抽象。

具体来说：

1. 起一个 `Next.js + shadcn/ui + Prisma` 项目
2. 跑通本地 PostgreSQL
3. 把你现有 Prisma schema 放进去
4. 先把“确认建档表单”页面做出来

为什么是这个顺序：

- 因为这是你整个系统最核心的人机交汇点
- 它能最快暴露字段是否过多、分组是否合理、AI 输出是否好接

### 如果你只想先做技术验证

那就只验证 3 件事：

1. 一条音频上传是否能稳定落盘
2. 一条 AI 解析结果是否能稳定回写 `interactions`
3. 一张确认表单是否能顺利写入 `customers`

只要这 3 件事通了，你这个项目就已经不是 PPT 了。

## 8. 一句话结论

如果你要进入真正开发阶段，我建议你先做：

`官方 Next.js 脚手架 + shadcn/ui + Prisma + PostgreSQL`

先把前端工作台和确认建档主链路跑通，再让 n8n、知识库、AI Chat、提醒系统逐步接进来。
